# JifeLog 로그인 페이지 PRD

## 문서 목적
- 현재 [components/pages/LoginPage.tsx](/Users/je/vscode/jifelog-project/components/pages/LoginPage.tsx)는 카드형 폼 중심의 로그인 화면이다.
- 이 문서는 2026년 기준 인증 화면 UX에 맞게 로그인 페이지의 화면 구조, 시각 톤, 입력 레이아웃을 재정의하기 위한 기획서다.
- 목적은 로그인 화면을 더 간결하고 넓게 보이게 만들고, 첫 인상과 입력 집중도를 동시에 높이는 것이다.

## 현재 구조 요약
- 어두운 배경 위에 중앙 카드가 놓여 있고, 카드 안에 소셜 로그인, 구분선, 이메일/비밀번호 입력, 로그인 버튼이 모두 배치되어 있다.
- 화면 자체보다는 카드가 강하게 인식되며, 모바일에서는 폭이 좁게 느껴진다.
- 전체 톤은 무난하지만 제품의 분위기를 충분히 살리는 구조는 아니다.

## 문제 정의
- 카드형 박스가 먼저 보이면서 화면이 조밀하고 답답해 보인다.
- 로그인은 빠르게 끝나야 하는데, 현재 구조는 입력보다 카드와 부가 UI의 존재감이 더 크다.
- 모바일에서 입력 폭이 좁아 보이고, 여백이 답답하게 느껴진다.
- 제품의 프리미엄하고 차분한 인상보다 일반적인 인증 모달에 가까운 느낌이 난다.

## 리디자인 목표
1. 로그인 화면을 카드형 폼이 아니라 화면 일체형 인증 화면으로 재구성한다.
2. 이메일과 비밀번호 입력이 가장 중요하게 보이도록 시각 우선순위를 정리한다.
3. 모바일에서도 입력 폭이 넓고 편안하게 느껴지도록 레이아웃을 개선한다.
4. 전체 톤은 JifeLog의 다크 기반 브랜드 인상과 자연스럽게 연결되도록 만든다.

## 유지 조건
- 유지 대상 입력 항목
  - 이메일
  - 비밀번호
- 유지 대상 액션
  - 로그인
  - 뒤로가기
  - 회원가입 링크
  - 비밀번호 찾기 링크
- 현재 로그인 로직 방향은 유지한다.

## 핵심 전략
### UX 전략 한 줄
로그인 화면은 설명보다 진입성과 집중이 먼저여야 한다.

### 구조 전략
- 큰 로그인 카드를 띄우는 대신, 배경과 입력 영역이 하나의 화면처럼 연결되는 구조를 사용한다.
- 사용자는 화면에 들어오자마자 바로 이메일과 비밀번호 입력을 이해할 수 있어야 한다.
- 로그인에 직접 필요 없는 장식성 구분 요소는 최소화한다.

## 트렌드 반영 방향
- 인증 화면은 모달처럼 보이기보다 독립된 제품 화면처럼 느껴져야 한다.
- 입력 필드는 넓고 가볍게 보이되 과한 박스 장식은 줄인다.
- 핵심 CTA는 단 하나만 강하게 두고, 보조 액션은 조용하게 배치한다.
- 텍스트는 짧고 선명해야 하며, 정보 밀도를 낮춰야 한다.

## Visual Thesis
짙은 다크 배경 위에 로그인 흐름이 화면에 자연스럽게 놓이는 구조. 박스보다는 여백과 타이포, 선형 입력 필드가 중심이 되며, 모바일에서도 넓고 정돈된 인상을 준다.

## Content Plan
1. Intro: 로그인 맥락과 제품 정체성을 짧게 전달
2. Input Area: 이메일, 비밀번호, 보조 링크
3. Action Area: 로그인 CTA와 회원가입 링크

## Interaction Thesis
- 첫 진입 시 제목과 입력이 부드럽게 등장한다.
- 포커스 상태에서는 입력선과 아이콘만 선명하게 반응한다.
- CTA는 hover/press에서만 짧고 확실하게 반응하고, 나머지 요소는 조용해야 한다.

## 타겟 사용자
- 이미 계정을 가진 기존 사용자
- 모바일에서 빠르게 로그인하려는 사용자
- 감성적인 화면은 좋지만 인증 과정은 즉시 이해되길 원하는 사용자

## 화면 구조 재정의
## 1. 상단 영역
### 목적
- 뒤로가기와 브랜드 톤을 조용하게 제공

### 요구사항
- 뒤로가기 액션은 유지
- 제품명 또는 작은 브랜드 라벨은 유지 가능
- 상단은 과하게 무겁지 않게, 입력 몰입을 방해하지 않도록 구성

## 2. 인트로 영역
### 목적
- 로그인이라는 행위의 맥락을 짧게 전달

### 요구사항
- 헤드라인은 1~2줄
- 설명문은 1문장 수준
- 예시 방향
  - `다시 돌아온 하루를 바로 이어가세요`
  - `저장해 둔 기록과 일정, 파일을 다시 불러옵니다`

## 3. 입력 영역
### 목적
- 이메일과 비밀번호 입력이 화면의 중심이 되게 만든다.

### 구조 원칙
- 큰 카드 사용 금지
- 입력은 충분한 폭을 갖고 화면 위에 직접 놓인 것처럼 보여야 한다.
- 필드는 박스형보다 선형 또는 매우 가벼운 면 처리 중심이 적합하다.
- 각 입력은 여백과 타이포로 구분한다.

### 구성 요소
1. 이메일 입력
2. 비밀번호 입력
3. `비밀번호를 잊으셨나요?` 링크

## 4. CTA 영역
### 목적
- 사용자가 망설이지 않고 로그인 액션을 선택하게 만든다.

### 요구사항
- `로그인` 버튼은 가장 강한 시각 요소
- 비활성/활성 상태가 명확해야 한다
- 회원가입 링크는 로그인 아래 조용한 보조 액션으로 둔다.

## 5. 소셜 로그인 처리
### 원칙
- 현재 소셜 로그인 버튼이 유지되어야 한다면, 메인 로그인 CTA보다 앞에 두지 않는다.
- 이메일 로그인 중심 구조를 해치지 않게 보조 옵션으로 낮춘다.
- 소셜 로그인은 상단 핵심 행동이 아니라 선택 가능한 부가 진입점으로 배치한다.

## 정보 구조
1. 뒤로가기
2. 브랜드 또는 라벨
3. 헤드라인
4. 설명문
5. 이메일 입력
6. 비밀번호 입력 + 비밀번호 찾기
7. 로그인 CTA
8. 회원가입 링크
9. 필요 시 소셜 로그인 보조 영역

## UI 구성 원칙
- 큰 메인 카드 사용 금지
- 카드 안에 또 다른 구분 박스 사용 금지
- 로그인 화면은 하나의 연속된 레이아웃처럼 보여야 한다.
- 구분은 선, 여백, 타이포로 해결한다.
- 모바일에서 입력 폭이 좁아 보이지 않도록 양옆 여백을 절제한다.

## 상태 설계
### 기본 상태
- 이메일/비밀번호 입력 대기

### 활성 상태
- 포커스된 필드의 선과 아이콘이 선명해짐

### 비활성 상태
- 로그인 버튼 비활성

### 오류 상태
- 향후 필요 시 이메일 또는 비밀번호 오류 문구가 입력 바로 아래에 표시될 수 있어야 한다.

## 카피 가이드
- 짧고 직접적인 문장 사용
- 감성은 배경 톤으로 전달하고, 문장은 기능적으로 유지
- 예:
  - `다시 오신 걸 환영합니다`
  - `로그인 후 기록하던 하루를 이어가세요`
  - `비밀번호를 잊으셨나요?`

## 비주얼 가이드
- 다크 배경과 teal 포인트 유지
- 배경은 깊이감 있게, 입력은 또렷하게
- 카드형 박스 대신 화면 일체형 구성
- 입력 필드는 넓고 정돈된 인상
- 버튼은 둥글고 명확하되 과도하게 부피감 있지 않게

## 레이아웃 가이드
### 데스크톱
- 한쪽에는 인트로 카피, 다른 한쪽 또는 중앙축에는 입력 영역 배치 가능
- 입력 영역은 독립 카드처럼 떠 있지 않아야 한다.

### 모바일
- 단일 컬럼
- 입력 폭을 최대한 확보
- 헤드라인, 입력, CTA가 한 흐름으로 자연스럽게 이어져야 한다.

## 모션 가이드
- 페이지 진입 시 제목과 입력이 짧게 등장
- 입력 포커스 상태는 색 변화 중심
- CTA hover/press는 절제된 반응만 사용
- 모션은 존재감이 아니라 완성도만 더해야 한다.

## 상세 요구사항
### 필수
- 카드형 로그인 박스 제거
- 이메일/비밀번호 입력 유지
- 로그인 버튼 우선순위 강화
- 회원가입 링크 유지
- 비밀번호 찾기 유지
- 모바일에서 입력 폭 확대

### 권장
- 소셜 로그인은 보조 옵션으로 낮추기
- 헤드라인과 설명문 더 짧게 정리
- 선형 입력 구조 사용

### 지양
- 큰 로그인 카드
- 과도한 구분선 장식
- CTA보다 소셜 버튼이 먼저 보이는 구조
- 좁은 폭의 모달 같은 구성
