# JifeLog 랜딩 페이지 PRD

## 문서 목적
- 현재 랜딩 페이지의 핵심 소개 영역은 [components/marketing/FeatureShowcase.tsx](/Users/je/vscode/jifelog-project/components/marketing/FeatureShowcase.tsx) 중심의 기능 나열형 구조다.
- 이 문서는 해당 구조를 2026년 기준 제품 랜딩 트렌드에 맞게 재정의하기 위한 기획 기준서다.
- 목적은 "기능 소개"보다 "브랜드 첫인상, 사용 장면, 전환 흐름"을 우선하는 랜딩 페이지를 설계하는 것이다.

## 문제 정의
- 현재 메시지 구조가 기능 단위로 분절되어 있어 첫 화면에서 제품 정체성이 강하게 각인되지 않는다.
- 제품이 제공하는 감정적 가치와 사용 맥락보다 개별 기능 설명이 먼저 노출된다.
- 섹션마다 시각적 리듬은 있으나, 전체 페이지를 관통하는 하나의 서사와 전환 설계는 약하다.
- "왜 지금 JifeLog를 써야 하는지"보다 "무슨 기능이 있는지"에 초점이 맞춰져 있다.

## 제품 정의 한 줄
JifeLog는 기록, 저장, 일정, 회고를 하나의 흐름으로 연결해 주는 개인 라이프 OS다.

## 리디자인 목표
1. 첫 화면 5초 이내에 브랜드 정체성과 제품 카테고리를 이해시킨다.
2. 사용자가 "예쁜 기록 앱"이 아니라 "일상 운영을 정리해 주는 통합 공간"으로 인식하게 만든다.
3. 기능별 상세 설명보다 실제 사용 장면과 결과 중심으로 설득한다.
4. 스크롤이 내려갈수록 기대감이 쌓이고, 마지막 CTA에서 회원가입 전환 의도가 강화되도록 한다.

## 타겟 사용자
### 1차 타겟
- 기록 습관은 있지만 여러 앱을 오가며 파편적으로 관리하는 20~30대 사용자
- 일정, 북마크, 사진, 메모를 한 곳에 모으고 싶은 생산성 지향 사용자
- 감성적 UI 선호와 실용성을 동시에 원하는 모바일 친화 사용자

### 2차 타겟
- 자기관리, 아카이빙, 디지털 정리에 관심 있는 크리에이티브 직군
- AI를 삶의 보조 도구로 자연스럽게 받아들이는 얼리어답터

## 핵심 가치 제안
- 기록은 예쁘게 남기고
- 정보는 잃지 않게 저장하고
- 일정은 흐트러지지 않게 정리하며
- 하루의 맥락은 AI와 함께 다시 해석한다

즉, JifeLog는 개별 도구의 집합이 아니라 "하루를 운영하는 감각적인 시스템"이어야 한다.

## 포지셔닝
### 지향점
- 감성 기록 앱의 무드
- 생산성 앱의 구조성
- AI 어시스턴트의 개인화 경험

### 피해야 할 방향
- 일반적인 SaaS 기능 카드 나열
- 너무 감성적이어서 실제 제품 구조가 약해 보이는 연출
- 지나치게 기술 제품처럼 보여 라이프스타일 제품의 매력이 약해지는 방향

## 랜딩 페이지 전략
### Visual Thesis
차분한 다크 톤 위에 티얼 포인트를 얹고, "하루의 장면들이 하나의 시스템으로 연결되는 느낌"을 큰 화면 구성과 깊이감 있는 모션으로 보여준다.

### Content Plan
1. Hero: 브랜드와 제품 정의를 단번에 각인
2. Support: JifeLog가 정리해 주는 하루의 흐름을 시각적으로 제시
3. Detail: 핵심 기능 4개를 "기능"이 아니라 "사용 시나리오"로 설명
4. Final CTA: 지금 시작해야 할 이유와 가입 액션을 단순하게 제시

### Interaction Thesis
- 첫 화면에서는 로고, 핵심 카피, 대표 앱 비주얼이 시간차로 등장한다.
- 스크롤 구간에서는 하루의 흐름이 이어지는 듯한 패럴랙스 혹은 스티키 전환을 사용한다.
- 기능 상세 구간은 hover보다 scroll reveal 중심으로, 정보가 "열리는" 인상을 준다.

## 핵심 메시지 구조
### 메인 메시지
당신의 하루를 기록하는 앱이 아니라, 정리하고 이어주는 공간

### 서브 메시지
피드, 북마크, 파일, 일정, AI 회고까지 흩어진 일상을 하나의 흐름으로 묶는다.

### 전환 메시지
복잡한 하루를 더 아름답고 선명하게 남기고 싶다면 지금 JifeLog에서 시작한다.

3. 제품 한 줄 정의 + 신뢰 장치
4. 하루의 흐름 섹션
5. 핵심 시나리오 섹션
6. AI 차별화 섹션
7. 가입 CTA
8. 푸터

## 섹션별 기획 요구사항

## 1. Header
### 목적
- 브랜드 인지와 즉시 행동 유도

### 요구사항
- 좌측에는 로고, 우측에는 `로그인`과 `시작하기`만 둔다.
- 스크롤 전에는 투명 또는 반투명, 스크롤 후에는 더 선명한 배경으로 전환한다.
- 메뉴 수를 늘리지 않는다. 랜딩 페이지에서 정보 탐색보다 전환을 우선한다.

## 2. Hero
### 목적
- 첫 화면에서 제품 정체성과 무드를 동시에 납득시킨다.

### 핵심 카피 방향
- 헤드라인은 2줄 이내
- 브랜드 또는 제품명은 가장 먼저 읽혀야 함
- 설명문은 1~2문장으로 제한

### 콘텐츠 요소
- 강한 메인 헤드라인
- 제품 정의 한 줄
- 주 CTA `무료로 시작하기`
- 보조 CTA `둘러보기`
- 대표 비주얼 1개

### 비주얼 방향
- 중앙 정렬보다 한쪽에 텍스트를 앵커로 두고 반대편에 대표 앱 비주얼을 크게 배치
- 기존 로고 단독 노출보다 실제 사용 장면이 느껴지는 제품 이미지 비중을 높인다
- 카드 여러 장을 띄우는 방식보다 한 개의 강한 비주얼과 제한된 보조 요소를 사용한다

### 성공 기준
- 첫 화면만 보고도 "기록 + 일정 + 저장을 아우르는 개인 관리 서비스"라는 인지가 가능해야 한다.

## 3. 제품 정의 + 신뢰 장치
### 목적
- 감성 랜딩으로만 보이지 않도록 구조적 신뢰를 더한다.

### 콘텐츠 방향
- 짧은 문장 2~3개로 제품의 범위를 명확히 설명
- 과도한 숫자 나열 대신 핵심 키워드 중심 정리

### 예시 키워드
- 기록
- 저장
- 일정
- 회고
- AI 보조

## 4. 하루의 흐름 섹션
### 목적
- 개별 기능이 아니라 사용자 하루의 흐름 안에서 제품을 이해시키기

### 핵심 메시지
기록하고, 모아두고, 정리하고, 다시 돌아보는 흐름이 끊기지 않는다.

### 구성 방식
- 아침, 낮, 밤 혹은 입력, 정리, 회고의 3단 구조
- 각 단계는 짧은 카피와 제품 화면 일부로 연결
- 스크롤 시 단계가 자연스럽게 이어지는 전환 사용

## 5. 핵심 시나리오 섹션
### 목적
- 현재 `FeatureShowcase`를 대체하는 본문 핵심 영역

### 원칙
- 기능명보다 사용자 효용이 제목이 되어야 한다.
- 4개 이내로 압축한다.
- 좌우 교차 레이아웃 반복만 쓰지 말고, 섹션마다 시각적 중심을 달리한다.

### 권장 시나리오
1. 흩어진 하루를 한 곳에 모은다
2. 저장한 정보가 다시 쓰이는 구조를 만든다
3. 일정과 기록이 분리되지 않게 이어진다
4. AI가 하루를 요약하고 다음 행동을 돕는다

### 기능 매핑
- Feed: 감성 기록과 일상 아카이빙
- Bookmark: 읽고 끝나는 정보가 아닌 다시 찾는 지식 보관
- Drive: 파일과 이미지의 안정적 저장
- Calendar: 일정과 기록의 연결
- AI: 하루 회고와 추천

## 6. AI 차별화 섹션
### 목적
- AI를 부가 기능이 아니라 브랜드 차별 포인트로 재정의

### 메시지 방향
- "AI 기능 있음"이 아니라 "하루를 해석해 주는 개인 어시스턴트"
- 정리, 회고, 추천, 감정 맥락화에 초점

### 비주얼 방향
- 대화 UI 단독보다, 사용자의 하루 데이터와 연결되는 맥락형 화면을 제안
- 미래지향적 과장보다 차분하고 신뢰감 있는 연출

## 7. Final CTA
### 목적
- 페이지 마지막에서 가입 결심을 단순하게 만든다.

### 요구사항
- 메인 문구는 짧고 선명해야 한다.
- 버튼은 1개의 주 액션, 1개의 보조 액션까지만 허용한다.
- 소셜 로그인은 제공 가능하되 시각적 우선순위는 이메일 시작보다 낮게 둔다.

### 권장 문구 톤
- 오늘부터 나만의 라이프 시스템을 시작하세요
- 기록하던 하루를 이제 연결하세요

## 카피 가이드
- 한국어 기준, 짧고 단정한 문장 사용
- 감성적인 표현은 허용하되 모호한 시어는 줄인다
- "혁신", "최고", "스마트한 솔루션" 같은 범용 마케팅 표현은 지양
- 사용자의 상태 변화가 읽히는 문장을 우선한다

## 비주얼 가이드
- 브랜드 메인 포인트 컬러는 현재 자산과 연결되는 teal 계열 유지
- 배경은 다크 슬레이트 기반, 필요 시 밝은 섹션을 제한적으로 섞어 대비 형성
- 둥근 카드 남발 금지, 큰 면과 여백 중심 구성
- 실제 앱 화면은 확대 크롭하여 사용하고 작은 썸네일 나열은 지양
- 로고는 장식이 아니라 브랜드 신뢰 장치로만 사용

## 모션 가이드
- 페이지 전체에서 2~3개의 명확한 모션 패턴만 사용
- 히어로 진입 모션
- 스크롤 기반 reveal 또는 sticky narrative
- CTA hover/press 피드백

### 금지 사항
- 모든 요소가 동시에 뜨는 과한 등장 애니메이션
- 카드마다 반복되는 개별 bounce/float
- 정보 전달과 무관한 장식성 모션

## 반응형 원칙
- 모바일에서 첫 화면 메시지와 CTA가 한 번에 보여야 한다.
- 모바일에서는 대형 비주얼을 유지하되 텍스트와 경쟁하지 않도록 순서를 재조정한다.
- 데스크톱 전용 복잡한 교차 레이아웃에 의존하지 않는다.

## KPI
- `Hero CTA 클릭률`
- `스크롤 50% 도달률`
- `회원가입 페이지 진입률`
- `모바일 이탈률`

## 구현 우선순위
1. Hero 재설계
2. `FeatureShowcase` 대체용 시나리오 섹션 설계
3. AI 차별화 섹션 강화
4. CTA 단순화 및 전환 문구 개선

## 범위 외

- 고객 후기/리뷰 섹션
- 블로그/콘텐츠 허브 연결

현재 단계에서는 브랜드 첫인상과 제품 이해도 개선이 우선이며, 사회적 증거 요소는 후속 실사용 데이터가 쌓인 뒤 추가한다.
